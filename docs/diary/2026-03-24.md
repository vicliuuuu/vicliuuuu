之前的方法我现在来看更像是一个模块堆叠，但是我觉得最后还是得升级到“统一表征+可学习决策+可预测未来”的这样的一个范式。我当前的状态其实是想做：CUT3R(几何)+OWL-ViT(语义)+TTC(安全)->MoE(融合)->控制，这样一个模块+融合的工作；有点就是的确有几何的认知，有开放的语义，有安全机制保障，具有可解释性，但是仅仅是拼接起来，但是没有一个完整的统一的调度。

现在广泛被提倡的端到端我觉得还是得有必要加入，我觉得今天最重要的一步就是应该要从信号拼接到状态建模，是否可以考虑讲多模态信号进行建模，给一个中间状态z?让这个系统从perception-driven system变成state-driven system? 此外，可以从“反应式控制”过渡到“可学习决策”，之前更像一个融合器，现在做一个策略分解，可以变成expert 1：追目标，expert 2：避障，expert 3：稳定控制。是这样做一个决策系统

量变引起质变，或许可以从“当前状态” → “未来预测(world model)”，之前是看到什么 → 做什么（reactive的反应式控制），是否可以有z_t + a_t → z_{t+1}的predictive（预测式）控制？这是进军world model的关键一步。

最后畅想一下，或许可以让语言加入，即实现VLA的功能？进入状态表示：z_t = f(geometry, semantics, TTC, text)，语言不止是找目标，而是定义任务，影响决策优先级；从而进入动态预测，z_t + a_t + text → z_{t+1}；进入规划层之后，plan = argmax trajectory under text constraint；这个是我畅想的VLA+ WAM

当然，还是要逐步来:

**level 1**: 还是要多模块融合 -> MoE -> action；

**level 2**: 实现轻度端到端, 多模块 → z → policy → action

**level 4**: VLA+WAM的全链路：(image + text) → z_t -> z_t + a_t → z_{t+1} → planning（受语言约束）→ action
